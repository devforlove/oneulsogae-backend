# 프론트 안내 — 결제완료 `GATHERING-007`(얼리버드 마감) 처리

> 대상: meeple-frontend 결제 연동 담당
> 관련 백엔드 변경: 2026-07-17 `feat(gathering): 결제완료 접수를 요청 상품 티어가로 확정하고 얼리버드 소진 거부` (commit `deaf9255`)

## 1. 무엇이 바뀌었나

결제완료 `POST /payments/v1/complete`가 **요청한 상품(productId)의 티어 가격으로 결제액을 확정**하도록 바뀌었습니다. 그 결과 **얼리버드 상품으로 결제하는데 그 사이 얼리버드가 소진되면 접수를 거부**합니다(신규 에러 `GATHERING-007`).

이전에는 결제 시점에 서버가 가격을 재계산해서, 체크아웃에서 본 얼리버드 금액과 실제 결제 금액이 조용히 달라질 수 있었습니다. 이제는 **"본 금액 = 결제 금액"** 을 보장하는 대신, 얼리버드가 마감된 경우엔 거부하고 프론트가 재조회하도록 합니다.

## 2. 언제 발생하나

- 유저가 모임 상세/체크아웃에서 **얼리버드 상품**(적용 티어 = `EARLY_BIRD`)을 보고 결제를 진행하는 중,
- 그 사이 다른 유저가 얼리버드 **마지막 수량을 소진**하면,
- `complete` 호출 시 **409 `GATHERING-007`** 이 반환됩니다.

얼리버드 한정 수량 경쟁에서만 발생하는 race 상황입니다. **정가(`NORMAL`)·할인가(`DISCOUNT`) 상품에는 발생하지 않습니다.**

## 3. 에러 응답 형태

- HTTP status: **409 Conflict**
- 본문(공통 응답 봉투):

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "GATHERING-007",
    "message": "얼리버드가 마감되었습니다. 최신 금액으로 다시 시도해주세요."
  }
}
```

분기는 **`error.code === "GATHERING-007"`** 로 판단하세요(HTTP 409는 다른 코드와 공유하므로 status만으로 구분 금지).

## 4. 처리 흐름 (권장)

```
complete 요청
  └─ 409 GATHERING-007 수신
       ├─ 1) "얼리버드가 마감되었습니다" 안내 (토스트/모달)
       ├─ 2) 모임 상세 재조회 → schedules[].productId · 금액 갱신
       │       (얼리버드 소진 후에는 DISCOUNT 또는 NORMAL productId·금액이 내려옴)
       └─ 3) 갱신된 금액을 사용자에게 다시 확인시킨 뒤 재결제 유도
```

**중요 — 자동 재요청 금지.** 얼리버드가 마감되면 금액이 **올라가므로**, 이전 화면 금액 그대로 조용히 재시도하면 안 됩니다. 반드시 상세를 재조회해 **바뀐 금액을 사용자에게 다시 확인**시킨 후 재결제하세요.

## 5. 바뀌지 않는 것 (그대로 사용)

- **요청 스키마 불변**: `POST /payments/v1/complete` 바디는 `{"productId": <number>}` 그대로입니다.
- **성공 응답 불변**: `{ "success": true, "data": { "amount": <number> } }` 그대로입니다.
- 프론트는 **모임 상세 응답의 `schedules[].productId`** 값을 그대로 넘기면 됩니다(이 값은 적용 티어 상품 id로, 얼리버드 유효→EARLY_BIRD / 소진→DISCOUNT or NORMAL이 자동 반영됨).
- **DB/DDL 변경 없음.**

## 6. 참고: complete가 반환할 수 있는 관련 에러 코드

| code | HTTP | 의미 | 프론트 처리 |
|---|---|---|---|
| `GATHERING-007` | 409 | **얼리버드 마감**(본 문서) | 상세 재조회 → 갱신 금액 재확인 후 재결제 |
| `GATHERING-004` | 409 | 성별 정원 마감 | "정원 마감" 안내, 재결제 불가 |
| `GATHERING-003` | 409 | 판매중(예정) 상태 아님 | "신청할 수 없는 일정" 안내 |
| `GATHERING-005` | 409 | 이미 신청(승인대기/참가) | "이미 신청함" 안내 |
| `GATHERING-006` | 404 | 상품 없음(productId 오류) | 상세 재조회 |
| `PAYMENTS-003` | 400 | 본인 성별 상품 아님 | 잘못된 productId — 상세 재조회 |
| `PAYMENTS-002` | 400 | 프로필 성별 미확정 | 온보딩(프로필) 완료 유도 |
